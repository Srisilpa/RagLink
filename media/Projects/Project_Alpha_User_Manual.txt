1. Introduction

This User Manual explains how to use the Project Alpha platform, SeriesTech’s enterprise AI
knowledge platform, for both end users submitting queries and tenant administrators managing
documents and users.

1.1 Who Should Read This Manual

-   End Users: Employees at a client organization who query the knowledge platform.
-   Tenant Admins: Users responsible for uploading documents, managing user roles, and
    reviewing analytics.
-   SeriesTech Customer Support: Staff assisting clients with onboarding and troubleshooting.

1.2 Definitions

  Term              Definition
  ----------------- ---------------------------------------------------------------------
  Query             A natural-language question submitted to the platform
  Citation          A reference to the source document and section supporting an answer
  Tenant Admin      A user role with permissions to upload documents and manage users
  Confidence Flag   An indicator shown when the system’s retrieval confidence is low

-----------------------------------------------------------------------------------------------

2. Getting Started

2.1 Logging In

Navigate to https://<tenant-name>.projectalpha.seriestech.com and log in using your
organization’s SSO credentials, or the email/password combination provided during onboarding if
SSO is not configured for your tenant.

2.2 Interface Overview

The main interface consists of three areas: the Query Bar at the top, the Conversation Panel in
the center showing your query history and answers, and the Source Panel on the right showing
citations for the currently selected answer.

Best Practice: Keep queries specific. “What is the notice period for fixed-term contract
employees?” retrieves more precise citations than “Tell me about contracts.”

-----------------------------------------------------------------------------------------------

3. Submitting Queries

3.1 Query Workflow

    User types a natural-language question in the Query Bar
            ↓
    System retrieves relevant document chunks
            ↓
    System reranks and generates an answer with citations
            ↓
    Answer displayed in Conversation Panel with linked sources in Source Panel

3.2 Example Query and Response

Example: A user asks, “How many days of Earned Leave can be carried forward?” The platform
returns an answer citing the HR Division Knowledge Base, Leave Matrix table, indicating the
carry-forward limit, along with a link to open the source document at the relevant section.

3.3 Understanding Confidence Flags

If the system cannot find strongly relevant content for a query, the answer is marked with a
Confidence Flag (a yellow indicator) and a note suggesting the user rephrase the query or
contact a subject matter expert. This corresponds to FR-206 in the Project Alpha Requirements
Specification.

3.4 Follow-Up Questions

The Conversation Panel supports follow-up questions within the same session, and the system
retains conversational context for up to 10 turns before requiring the user to start a new
session.

4. Tenant Admin Functions

4.1 Uploading Documents

1.  Navigate to Admin > Documents > Upload.
2.  Select one or more files (PDF, DOCX, or TXT, up to 50 MB each).
3.  Assign a category tag (for example, “HR Policy,” “Engineering Standards,” “Product
    Manual”).
4.  Click Upload and monitor ingestion status on the Documents dashboard.

4.2 Document Status Table

  Status       Meaning                                     Action Needed
  ------------ ------------------------------------------- ------------------------------
  Queued       Document is waiting for processing          None
  Processing   Text extraction and embedding in progress   None
  Indexed      Document is searchable                      None
  Failed       Extraction or embedding failed              Re-upload or contact Support

Warning: Documents with scanned images and no extractable text layer will fail ingestion. Run
the document through OCR before uploading, or contact SeriesTech Support for assistance.

4.3 Managing Users and Roles

  Role           Permissions
  -------------- -------------------------------------------------------------
  Tenant Admin   Full access: upload documents, manage users, view analytics
  Uploader       Can upload and tag documents, cannot manage users
  Viewer         Can submit queries only

Navigate to Admin > Users > Invite User to add a new user, specifying their role from the table
above.

4.4 Reviewing Analytics

The Admin > Analytics dashboard shows query volume over time, top queries, average response
time, and documents most frequently cited in answers — useful for identifying gaps in the
document set.

6. Frequently Asked Questions

Q: Can Project Alpha answer questions using information outside our uploaded documents? A:
No. By design, Project Alpha only generates answers grounded in retrieved content from your
organization’s uploaded documents, to avoid ungrounded or fabricated responses.

Q: How often should we re-upload updated documents? A: Whenever a source document changes
materially. The platform supports re-ingestion without duplicating chunks, as described in
FR-106 of the Requirements Specification.

Q: Is our data visible to other tenants? A: No. Each tenant’s documents and embeddings are
stored in an isolated ChromaDB collection, as described in the Technical Design Document,
Section 4.2.

Q: Can we choose between on-premises and cloud-hosted generation? A: Yes. Tenants may opt for
on-prem generation using Llama models served via Ollama, or cloud-hosted generation using
OpenAI APIs, depending on data residency requirements.

Q: What file formats are supported for upload? A: PDF, DOCX, and TXT, up to 50 MB per file.

6A. Personal Settings

6A.1 Notification Preferences

Users can configure email notifications for ingestion completion (useful for Tenant Admins
uploading large document batches) and weekly usage digests under Settings > Notifications.

6A.2 Query History

The History tab retains a searchable log of a user’s past queries and answers for 90 days,
after which entries are archived per the Data Retention rules described in the IT Security &
Infrastructure Guide (STL-IT-SEC-002), Section 8.3.

6B. Best Practices for Getting Better Answers

-   Phrase queries as complete questions rather than fragments; “What is the escalation path
    for a P1 incident?” performs better than “escalation P1.”
-   When an answer seems incomplete, ask a targeted follow-up rather than re-asking the same
    question, since the platform retains conversational context (Section 3.4).
-   If your organization has multiple versions of a policy document, ensure outdated versions
    are archived rather than left indexed alongside the current version, to avoid conflicting
    citations.
-   Use the category tags applied during upload (Section 4.1) to help Tenant Admins and Support
    narrow down retrieval issues when reporting a ticket.

6C. Example Walkthrough — Tenant Admin Onboarding a New Department

Example: A client’s IT department wants their internal runbooks searchable alongside existing
HR and Engineering documents. The Tenant Admin uploads the runbook PDFs under a new “IT
Runbooks” category tag (Section 4.1), monitors ingestion status until all documents show
“Indexed” (Section 4.2), and then reviews the Analytics dashboard (Section 4.4) after one week
to confirm the new documents are being retrieved for relevant queries.
